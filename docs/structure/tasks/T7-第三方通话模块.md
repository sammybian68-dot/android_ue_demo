## 开发步骤

### Step 1: 阅读基准画板
读取 `37-第三方-来电展开.json` 和 preview 截图：
- 当前通话方信息（上方）
- 新来电方信息（下方）
- 分隔线 `bt_line_3 .png`
- 接听 `keyboard_btn_number_answer_n.png` + 挂断 `keyboard_btn_number_hang_up_n.png`
- 挂断当前通话 `bt_btn_hang_up_1_n.png`
- 缩小按钮 `bt_btn_lessen_n.png`

### Step 2: 搭建第三方来电全屏布局
- 上半部分：当前通话方号码/姓名 + 通话时间 + 挂断当前按钮
- 分隔线
- 下半部分：新来电号码/姓名 + 接听/挂断按钮

### Step 3: 搭建第三方通话全屏布局
读取 `38-第三方-通话展开.json`：
- 两路通话信息分上下显示
- 底部功能按钮：切换 `keyboard_btn_cut_n.png` + 键盘 + 静音 + 挂断 + 切换通话

### Step 4: 搭建第三方来电/通话小窗
读取 `39-第三方-来电小框.json` 和 `40-第三方-通话小框.json`：
- 小窗背景 `phone_pop_up_2.png`（双行小窗）
- 分隔线 `phone_pop_up_line.png`
- 来电小窗：接听 `bt_btn_answer_n.png` + 挂断
- 通话小窗：切换 `bt_btn_cut_n.png` + 静音 + 挂断

### Step 5: 实现通话切换逻辑
- 接听第三方来电 → 当前通话保持
- 切换按钮 → 在两路通话间切换
- 挂断 → 断开一方，回到单通话

### Step 6: 验证
对比四个 preview 截图，确认全屏和小窗两种形态的布局还原度。
