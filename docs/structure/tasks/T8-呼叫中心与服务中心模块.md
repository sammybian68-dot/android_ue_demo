## 开发步骤

### Step 1: 呼叫中心入口页
读取 `35-5-呼叫中心.json`：
- 全屏显示服务说明文字
- 拨打按钮 `keyboard_btn_number_answer_n.png`
- 缩小按钮 `bt_btn_lessen_n.png`

### Step 2: I-Call 客服中心
读取 `41-5-客服中心（I-Call）.json` 系列：
- 入口页：服务描述 + 拨打按钮
- 拨号中全屏（42）：号码 + 挂断
- 拨号中小窗（43）：`phone_pop_up_1.png` + 挂断
- 通话中全屏（44）：号码 + 通话时间 + 挂断
- 通话中小窗（45）：小窗 + 挂断

### Step 3: E-Call 服务中心
读取 `46-6-拨打服务中心（E-Call）.json` 系列：
- 拨号中全屏（46）：紧急呼叫提示 + 缩小按钮
- 拨号中小窗（47）：`phone_pop_up_ecall_1.png`（E-Call 专属小窗背景）
- 通话中全屏（48）：通话时间 + 缩小按钮
- 通话中小窗（49）：`phone_pop_up_1.png` + 挂断

### Step 4: 抽取通话基类
I-Call 和 E-Call 的全屏/小窗模式与普通通话类似，可抽取基类：
- BaseCallActivity：全屏/小窗切换逻辑
- BaseCallMiniWindow：小窗浮窗管理

### Step 5: 导入专属切图
- `phone_pop_up_ecall_1.png` — E-Call 小窗背景
- `keyboard_btn_number_answer_n.png` — 拨打按钮

### Step 6: 验证
对比 11 个 preview 截图，确认各页面布局还原度。
